DoliProspectForm — Guide utilisateur

Ce guide présente le module du point de vue de l’utilisateur final : à quoi il sert, comment l’utiliser au quotidien et où trouver les principales actions dans Dolibarr.

À quoi sert ce module ?

DoliProspectForm permet de proposer à vos prospects des formulaires publics accessibles sans connexion à Dolibarr : ils peuvent saisir leurs coordonnées et joindre des documents (par exemple des factures énergie au format PDF), directement depuis un lien que vous leur envoyez.

Particulier

Personne physique, identité, adresse et pièces jointes.

Professionnel

Entreprise (recherche par raison sociale ou identifiants lorsque disponibles), adresse du siège, contact et pièces jointes.

Une page d’aiguillage peut être proposée : le prospect choisit le type de formulaire adapté avant de continuer. Les données alimentent votre Dolibarr : fiche tiers, contact et fichiers sur la fiche, selon le scénario.

Qui fait quoi dans l’organisation ?

RôleUsage typique
AdministrateurActive le module, configure les formulaires, textes, notifications et liens « anonymes » si besoin.
Commercial / métierUtilise les liens signés ou les invitations, consulte les soumissions.
Prospect (externe)Ouvre le lien, remplit le formulaire, envoie les pièces, sans compte Dolibarr.

Premiers pas après activation

  1. Textes par défaut des pages publiques prêts à l’emploi ; personnalisables dans la configuration du module.
  2. Des modèles d’e-mail utiles peuvent être créés automatiquement ; à retrouver dans l’administration des modèles Dolibarr.
  3. Vérifier l’envoi d’e-mails Dolibarr et les notifications de nouvelles soumissions.

Où trouver le module dans Dolibarr ?

Le module apparaît dans le menu principal sous une entrée dédiée (libellé du type « DoliProspectForm » selon la traduction).

  • Tableau de bord — vue d’ensemble et suivi
  • Invitation — parcours guidé vers l’e-mail prospect
  • Formulaires renseignés — historique des soumissions

Les réglages avancés (activation des formulaires, durée des liens, utilisateur technique, textes, modèle de notification, e-mail de secours, etc.) se trouvent dans la configuration du module.

Faire compléter un dossier par un prospect

Depuis une fiche tiers, des balises de substitution peuvent insérer des liens vers la page d’aiguillage, le formulaire particulier ou professionnel. Le bon commercial est associé au dossier selon le contexte.

Votre administrateur indique les balises à utiliser (écran d’aide ou configuration).

Pour un site web ou une campagne générique, un lien public stable peut être affiché dans la configuration. Sur l’aiguillage, le visiteur peut indiquer l’e-mail d’un consultant Dolibarr si l’option est proposée.

Création d’un tiers prospect avec libellé provisoire, e-mail optionnel, puis ouverture de l’écran d’envoi de message Dolibarr. Le lien reçu permet de compléter la fiche créée.

Côté prospect : déroulement du formulaire

1
Ouvre le lien (e-mail, SMS, site web…).
2
Sur l’aiguillage, choix Particulier ou Professionnel.
3
Champs obligatoires et PDF demandés.
4
Formulaire pro : recherche d’entreprise possible ; le prospect valide et complète le contact.
5
Captcha si activé sur votre instance.
6
Message de remerciement après envoi. Lien expiré ou formulaire désactivé : message invitant à contacter votre entreprise.

Suivi des demandes reçues

La liste des formulaires renseignés affiche type, date, tiers, nombre de pièces, selon vos droits.

Les notifications e-mail peuvent prévenir le commercial identifié, sinon une adresse de secours définie par l’administrateur. Le contenu dépend du modèle d’e-mail choisi.

Bonnes pratiques

  • Indiquez clairement quels documents joindre (textes d’aide en configuration).
  • Testez le parcours complet après un changement de configuration.
  • Traitez les liens comme des accès sensibles.
  • Lien expiré : demandez la durée configurée et renvoyez une nouvelle invitation.
